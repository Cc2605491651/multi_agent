# structured-output

## 何时触发
本技能由汇总 / summarize / 收尾类节点强制启用，目标是让产出有清晰层级。

## 必须遵守
- 用 Markdown 一级标题 `#` 作为最外层 章节
- 关键结论用「结论：…」前缀放在每个章节末尾
- 数据点用列表呈现，避免长段落
- 如有数字/价格/时间，必须放在表格里

## 反例（不要这样写）
> 经过我们大量调研，整体看 A 工具不错。B 略贵但功能强。C 是 SaaS 便宜但本地不行。
> 综上推荐 A。

## 正例
```
# 选型决策
- 入选：A 工具
- 备选：B 工具
- 落选：C 工具（不满足本地部署硬约束）

| 工具 | 年成本 | 部署 |
|---|---|---|
| A | 30 万 | 本地 |
| B | 40 万 | 本地 |
| C | 20 万 | SaaS |

结论：选 A。年成本 30 万落在 50 万预算内，且符合本地部署硬约束。
```
